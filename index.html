本系统实时监控飞鹰策略、单卖策略、双卖策略三类期权持仓的风险指标，自动判定风险等级并生成调仓建议。

指标正常关注预警高危
保证金使用率≤60%60-80%80-90%>90%
IV分位30-70%70-80%80-90%>90%

三、字段释义

  • Delta: 标的价格变动1元，组合价值变动金额
  • Gamma: Delta对标的价格的敏感度
  • Vega: IV变动1%，组合价值变动金额
  • Theta: 时间流逝1天，组合价值变动金额
  • IV分位: 当前IV在历史分布中的百分位
  • DTE: 到期剩余天数

一、系统概述

本系统为飞鹰/单卖/双卖策略一体化持仓风控系统，覆盖国内6大期货交易所14个商品期权品种，提供实时希腊字母计算、风险等级判定、调仓建议和Web可视化界面。

核心功能

  • 手动维护持仓数据（positions.json），系统自动计算风控指标
  • 三级标的价格来源：TqSdk期货行情 [R] > IV分位接口 [S] > 行权价均值 [U]
  • 两级期权价格来源：TqSdk期权行情 [R] > positions.json静态价格 [S]
  • 希腊字母计算：TqSdk OPTION_GREEKS引擎 [R] > Black-Scholes公式 [S]
  • 风险等级综合判定（保证金率 / IV分位 / ATR偏离度 / DTE）
  • Web界面实时展示（Bloomberg风格深色主题）
  • 盘中自动刷新（默认10分钟）

六、计算公式详解（核心章节）

6.1 DTE（距到期天数）计算

优先级1（最高）：expiry_date字段

exp_d = datetime.date.fromisoformat("2026-06-11")
dte = max((exp_d - datetime.date.today()).days, 1)

直接读取positions.json中的expiry_date字段；今天到期时DTE=1（避免BS公式T=0发散）。

优先级2（fallback）：_estimate_dte()估算

# 按"合约月前一个月15日"估算
# 例："2607" → 2026年6月15日
# 例外：JD合约到期日 = 合约月当月15日（不往前推一个月）

仅当expiry_date字段为空时触发，精度较低，建议手动填写expiry_date。

6.2 IV（隐含波动率）估算

函数：_estimate_iv(S, K, T, is_call, iv_data=None, product=None)

优先级来源说明
0（最高）TqSdk OPTION_IMPV引擎 [R]从市场期权价格反推IV，精度最高
1IV分位接口数据 [S]iv_avg字段 / 100
2历史波动率（HV）[S]本地数据库30日历史波动率
3从平值期权价格反推 [S]简化公式：iv ≈ 2 × option_price / S × sqrt(T)
4（最低）默认值0.15（15%）

6.3 希腊字母（Greeks）计算

方向乘数规则：

持仓方向multiplier说明
卖出期权（空）-1卖方Greeks取反
买入期权（多）+1买方Greeks取正

Black-Scholes公式（优先级2 fallback）：

// 参数
S = 标的价格（期货价格）
K = 行权价
T = DTE / 365.0   // 年化到期时间
iv = 隐含波动率（小数）
r = 0.03   // 无风险利率（3%）

// d1, d2
d1 = [ln(S/K) + (r + iv²/2) × T] / (iv × √T)
d2 = d1 - iv × √T

// N(x) = 标准正态分布累积函数
// N'(x) = e^(-x²/2) / √(2π)   // 标准正态概率密度函数

Delta（Δ）— 标的价格变动敏感度：

期权类型买方Delta卖方Delta（× -1）
CallN(d1)-N(d1)
PutN(d1) - 11 - N(d1)

Gamma（Γ）— Delta变化率（加速度）：

Gamma = N'(d1) / (S × iv × √T)
⚠️ 深度虚值+临近到期：N'(d1) → 0，Gamma → 0（BS模型固有限制）

Vega（ν）— 波动率变动敏感度：

Vega = S × N'(d1) × √T / 100

Theta（θ）— 时间衰减：

// Call
theta_call = [-S×N'(d1)×iv/(2×√T) - r×K×e^(-rT)×N(d2)] / 365

// Put
theta_put = [-S×N'(d1)×iv/(2×√T) + r×K×e^(-rT)×N(-d2)] / 365
⚠️ BS模型在T→0时数值不稳定：临近到期时N'(d1)→0，第一项消失，卖方Theta符号可能翻转（显示为负数，与实际相反）。
DTE ≤ 2天的Theta仅供参考，不构成调仓依据。

持仓级 vs 每手 Greek：

显示方式数值说明
持仓级delta × volume × multiplier系统当前显示方式
每手delta（未乘手数）文华财经等软件显示方式

6.4 浮动盈亏（Float PnL）计算

当前实现：从positions.json直接读取 float_pnl 字段（手动填写，来自交易软件）。

验证公式（参考）：

// 卖出期权
float_pnl = (open_price - last_price) × volume × 期货公司保证金乘数

// 买入期权
float_pnl = (last_price - open_price) × volume × 期货公司保证金乘数

6.5 ATR偏离度计算

atr_deviation = (current_price - pre_settlement) / atr
参数来源说明
current_priceTqSdk期货实时价 [R] 或 IV API [S]标的当前价格
pre_settlementTqSdk quote.pre_settlement [R]昨结算价（今日结算参考）
atrIV分位接口 atr 字段 [S]平均真实波幅（14日）

6.6 IV价差（iv_spread）计算

// 对每个品种，找最近月和最远月的IV，计算差值
for product in strategy_positions:
    expiries = {pos["expiry"]: pos["iv"] for pos in positions if iv > 0}
    if len(expiries) >= 2:
        near_iv = expiries[min(expiries)]
        far_iv = expiries[max(expiries)]
        iv_spread_values.append(far_iv - near_iv)

iv_spread = mean(iv_spread_values)   // 策略级平均

6.7 风险等级综合判定

综合4个维度，按"最差原则"定级：

维度正常（1）关注（2）预警（3）紧急（4）
保证金使用率≤60%>60%>80%>90%
IV分位≤70%>70%>80%>90%
ATR偏离度（绝对值）≤1.0>1.0>1.5>2.0
最小DTE≥5天<5天<3天<1天

十二、常见问题（FAQ）

Q1：为什么DTE≤2天的期权Greek数值异常？

A：BS模型在T→0时数学特性不稳定：Gamma → 0（N'(d1)→0），Theta符号可能翻转（利率项主导）。

建议：DTE≤2天不依赖Greek数值，以到期盈亏和是否行权为决策依据。

Q2：系统显示的Theta和文华财经不一致？

A：两个可能原因：1. 持仓级 vs 每手：本系统显示持仓级Greek（已乘手数和方向），文华显示每手Greek；2. IV来源不同：本系统用TqSdk/BS计算，文华用自家IV。

Q3：如何判断Greeks计算是否正确？

A：三步验证法：1. 方向检查：卖出期权delta应为负，gamma/vega应为负，theta应为正；2. 数量级检查：delta绝对值≤1（持仓级可>1），theta单位：元/天；3. 交叉验证：与文华/同花顺等软件对比每手Greeks。

Q4：为什么有些持仓的iv=0？

A：以下情况会导致IV=0：1. TqSdk未获取到期权价格（免费版15分钟延迟，可能返回0）；2. IV分位接口未覆盖该品种；3. BS公式fallback时所有IV估算方法均失败（极少发生）。

Q5：如何添加新的交易品种？

A：三步操作：1. 更新EXCHANGE_MAP：在risk_system.py第84-112行添加品种→交易所映射；2. 更新PRODUCT_NAME_MAP：添加品种中文名称；3. 更新config.ini：在[strategy_mapping]段添加品种→策略映射。

Q6：服务启动后浏览器打不开？

A：排查步骤：1. 检查端口是否被占用（netstat -ano | findstr 18099）；2. 检查python进程是否running（tasklist | findstr python）；3. 手动测试HTTP服务（curl http://localhost:18099/）；4. 查看risk_system.py启动日志（控制台窗口）。
